= 使用NetApp Data Classification掃描資料來源
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
NetApp Data Classification 會掃描您所選擇的儲存庫（包括磁碟區、資料庫結構或其他使用者資料）中的資料，以識別個人和敏感資料。Data Classification 接著會對您的組織資料進行對應、分類每個檔案，並識別資料中的預定義模式。掃描的結果是一份包含個人資訊、敏感個人資訊、資料類別和檔案類型的索引。

初步掃描後，資料分類將以循環方式持續掃描您的資料以偵測增量變化。這就是為什麼保持實例運行很重要。

您可以在磁碟區層級或資料庫模式層級啟用和停用掃描。



== 了解掃描類型

您可以在資料分類中進行兩種類型的掃描：

* **僅對應掃描** 僅提供資料的高階概觀，並且僅針對選定的資料來源執行。由於僅對應掃描無需存取檔案來查看其中的資料，因此比完整掃描耗時更短。
* **完整掃描** 可對您的資料進行深度掃描。完整掃描包括映射掃描和檔案內部資料分類。



TIP: 執行僅對應掃描有助於取得資料概觀，並找出要執行完整掃描的特定區域。

{asterisk} Data Classification 對掃描的資料量沒有限制。每個 Console 代理程式支援掃描和顯示 500 TiB 的資料。若要掃描超過 500 TiB 的資料、link:https://docs.netapp.com/us-en/console-setup-admin/concept-connectors.html#connector-installation["安裝另一個控制台代理"^]然後link:task-deploy-overview.html["部署另一個資料分類實例"]。+ Console UI 會顯示來自單一連接器的資料。如需檢視來自多個 Console 代理程式之資料的秘訣、請參閱link:https://docs.netapp.com/us-en/console-setup-admin/task-manage-multiple-connectors.html#switch-between-connectors["使用多個控制台代理"^]。
